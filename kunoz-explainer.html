Security & Data

Enterprise-Grade Data Protection

Every query goes through Row Level Security. Every action is audited. Every role boundary is enforced server-side.

🔒
Row Level Security (RLS)

Every database table has RLS policies. Employees only see their own data. Supervisors see their site. HR sees their scope. Enforced at the Postgres level — can't be bypassed from the client.

🛡
Server-Side Auth

All data mutations use Next.js Server Actions with Supabase server-side authentication. The user's identity is always derived from the auth session — never from client-side data.

📝
Complete Audit Trail

Every HR action (corrections, approvals, warnings, leave) is logged with: who performed it, when, what changed (old vs. new values), and why (mandatory reason field).

🌐
GPS Anti-Spoofing

GPS readings are validated for null-island detection (0,0 coords), accuracy-zero spoofing, and unreliable readings (>500m accuracy). Suspicious readings are flagged.

✅
Input Validation

All inputs validated with Zod schemas on the server. No raw user input reaches the database. Form data, GPS coordinates, and action parameters are all strictly typed and validated.

👥
Registration Approval

New users can't access the system until an admin approves their registration. Rejected users are locked out. This prevents unauthorized access from self-registration.
