Offline-First

Works Without Internet

Construction sites have unreliable connectivity. Kunoz doesn't just handle it — it's built for it. Every clock-in works offline and syncs seamlessly when the connection returns.

📱

Clock-In Offline

Employee taps clock-in. No internet detected. GPS and timestamp captured locally.

💾

Stored in IndexedDB

Record saved to the device's local database with a unique session ID. Pending counter shows in the UI.

🔄

Auto-Sync Engine

When connectivity returns, the sync engine replays pending actions in order. Clock-ins first, then clock-outs.

☁

Server Confirmed

Records validated and stored on the server. Employee gets a sync confirmation notification. Conflicts resolved automatically.

⚠

Conflict Resolution

If the server already has a record (e.g., supervisor batch-checked the employee in), the duplicate is detected and treated as synced. Server timestamps always win. Failed syncs are shown in a review dialog where employees can retry or discard.

Security & Data

Enterprise-Grade Data Protection

Every query goes through Row Level Security. Every action is audited. Every role boundary is enforced server-side.

🔒
Row Level Security (RLS)

Every database table has RLS policies. Employees only see their own data. Supervisors see their site. Enforced at the Postgres level.

🛡
Server-Side Auth

All data mutations use Next.js Server Actions with Supabase server-side authentication. User identity derived from auth session.

📝
Complete Audit Trail

Every HR action is logged with: who, when, what changed, and why. Mandatory reason field for all corrections.

🌐
GPS Anti-Spoofing

GPS readings validated for null-island detection, accuracy-zero spoofing, and unreliable readings (>500m accuracy).

✅
Input Validation

All inputs validated with Zod schemas on the server. No raw user input reaches the database.

👥
Registration Approval

New users can't access the system until an admin approves their registration.

Step 2: Walk Through Each Scenario

1

Registration & Approval Flow

Admin
  1. Open app in incognito → go to /login
  2. Click "Create account" → sign up with a new email
  3. Check email → verify the link works
  4. Log in → you should see the "Pending Approval" page
  5. Log in as Admin → go to /admin/users
  6. Approve the new user and assign a role
  7. Log back in as the new user → they should see the dashboard
Expected: Unapproved users are blocked. Approved users see role-appropriate dashboard.
2

Employee Clock-In (Online)

Employee
  1. Log in as worker@kunoz-test.com
  2. Go to /dashboard/attendance
  3. Allow GPS permission when prompted
  4. Tap the "Clock In" button
  5. Verify the success message shows location name + time
  6. Check that attendance status shows "Present" or "Late"
  7. Wait, then tap "Clock Out"
Expected: Record created with GPS coords. Status auto-computed against shift time. History updated.
3

Offline Clock-In & Sync

Employee
  1. Log in as employee → load the attendance page
  2. Open DevTools → Network → toggle Offline
  3. Verify the offline indicator appears (red dot / banner)
  4. Tap "Clock In" — should still succeed
  5. Check the pending sync counter in the UI
  6. Re-enable network → watch for auto-sync
  7. Verify the record appears on the server (check HR dashboard)
Expected: Clock-in succeeds offline. Pending count shows. Auto-syncs when online. Record appears in HR view.
4

Supervisor Batch Clock-In

Supervisor
  1. Log in as supervisor@kunoz-test.com
  2. Go to /dashboard/bulk-checkin
  3. Select 3+ employees from the crew list
  4. Tap "Clock In Selected"
  5. Verify each employee gets a separate attendance record
  6. Check records show method = supervisor_batch
  7. Verify late/on-time status is computed per employee's shift
Expected: N records created in one action. Supervisor's GPS used. Method tagged for audit.
5

Site Attendance Dashboard

Supervisor
  1. Log in as supervisor → go to /dashboard/site-attendance
  2. Verify you see only your site's employees
  3. Check the present / absent / late counts are correct
  4. Add a note to an attendance record
  5. In another browser, clock in as an employee at the same site
  6. Check if the dashboard updates (may need refresh)
Expected: Scoped to site only. Counts accurate. Notes saved. New check-ins reflected.
6

HR Reports & Excel Export

HR / Admin
  1. Log in as hr@kunoz-test.com (or admin)
  2. Go to /dashboard/reports
  3. Select "Daily Attendance" report → pick today's date
  4. Verify data shows for all sites
  5. Click "Export to Excel" → download the .xlsx file
  6. Open in Excel → check columns match the on-screen data
  7. Test at least 3 different report types
Expected: Reports show correct data. Excel files download and open correctly with proper formatting.

  1. Go to /dashboard/hr-actions
  2. Correct an attendance record → change clock-in time
  3. Check the audit log shows old/new values + your name
  4. Approve an overtime entry
  5. Issue a warning to an employee
  6. Mark an employee as on leave for today
Expected: All actions succeed. Audit trail complete. Status re-computed after corrections.

Shift Management

10

PWA Install & Mobile Test

Any Role
  1. Open the app on a mobile phone (Chrome or Safari)
  2. Check for "Add to Home Screen" prompt or install banner
  3. Install the PWA → open from home screen
  4. Verify it launches in standalone mode (no browser bar)
  5. Navigate between pages → check bottom navigation works
  6. Toggle airplane mode → verify cached pages still load
Expected: Installs as app. Standalone mode. Pages cached. Navigation works offline.
11

Role-Based Access Verification

All Roles
  1. As Employee: try to access /admin/users → should redirect
  2. As Employee: verify no sidebar links to admin pages
  3. As Supervisor: verify no access to HR actions or admin
  4. As HR: verify access to reports but not admin/users
  5. As Admin: verify full access to everything
Expected: Each role only sees their allowed pages. Direct URL access redirects unauthorized users.
12

Geofence Violation Test

Employee + HR
  1. In Chrome DevTools: Sensors tab → set custom GPS
  2. Set coordinates far from any Kunoz site (e.g., 0, 0)
  3. Log in as employee → tap "Clock In"
  4. Check if a warning is shown (outside geofence)
  5. Log in as HR → check for geofence violation notification
  6. Check that the record is flagged but not blocked
Expected: Clock-in succeeds but flagged. HR notified. Record shows GPS mismatch.
